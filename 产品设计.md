# IoT Platform 产品功能介绍

> ⚠️ 这是一个实验性项目，采用 vibe coding 方式开发。

## 1. 产品概述

IoT Platform 是一个轻量级的物联网平台，使用 Go 语言开发，集成了设备管理、MQTT 消息服务、实时 WebSocket 通信、数据存储和 Web 可视化仪表盘。

**固件限制**:

### 3.2 固件升级 - 灰度发布 (Firmware Upgrade)

**模块**: `internal/firmware/`

**功能**:
- 按百分比随机选择设备进行升级
- 灰度发布策略 (支持百分比 1-100)
- 升级任务管理
- 扩量升级 (分批次提升百分比)
- 取消升级任务
- 重试失败设备
- 设备固件状态跟踪

**升级流程**:
```
1. 选择固件 → 系统自动识别设备类型
2. 设置升级策略 (全量100% / 灰度百分比)
3. 系统按百分比随机选择目标设备
4. 创建升级任务，MQTT通知设备
5. 设备下载固件并上报状态
6. 可在任务详情查看进度、扩量、取消
```

**扩量流程**:
```
15% → 50% → 100% (分批次确认后逐步扩量)
```

**主要接口**:
| 方法 | 说明 |
|------|------|
| `CreateUpgradeTaskByPercentage(firmwareID, percentage)` | 创建灰度升级任务 |
| `ExpandTask(taskID, newPercentage)` | 扩量升级 |
| `CancelTask(taskID)` | 取消任务 |
| `RetryFailedDevices(taskID)` | 重试失败设备 |
| `GetUpgradeTask(taskID)` | 获取任务详情 |
| `ListUpgradeTasks()` | 列出任务 |

**数据表**:
| 表名 | 说明 |
|------|------|
| `upgrade_tasks` | 升级任务表 |
| `upgrade_task_devices` | 升级任务设备明细 |

**MQTT 主题**:
| 主题 | 方向 | 描述 |
|------|------|------|
| `devices/{id}/firmware/info` | 平台→设备 | 固件升级通知 |
| `devices/{id}/firmware/status` | 设备→平台 | 升级状态上报 |

**升级状态**:
| 状态 | 说明 |
|------|------|
| `pending` | 等待升级 |
| `downloading` | 下载中 |
| `installing` | 安装中 |
| `success` | 升级成功 |
| `failed` | 升级失败 |

### 3.3 告警规则引擎 (Alert Rule Engine)

**模块**: `internal/alert/`

**功能**:
- 告警规则 CRUD (创建/读取/更新/删除)
- 规则启用/禁用控制
- 多种条件类型支持 (阈值/状态/表达式)
- 多种动作执行 (WebSocket/Webhook/MQTT/Dashboard)
- 实时告警通知
- 告警确认/解决管理
- 冷却时间控制 (防止重复告警)

**条件类型**:
| 类型 | 说明 | 示例 |
|------|------|------|
| `threshold` | 阈值条件 | temperature > 30 |
| `status` | 设备状态 | status == "offline" |
| `change` | 变化检测 | (暂不支持) |
| `expression` | 表达式条件 | temperature > 25 and humidity > 50 |

**动作类型**:
| 类型 | 说明 |
|------|------|
| `websocket` | 通过 WebSocket 推送实时告警 |
| `webhook` | 发送 HTTP POST 请求到指定 URL |
| `mqtt` | 发布 MQTT 消息到指定主题 |
| `dashboard` | 在 Dashboard 中显示告警 |

**优先级**:
| 优先级 | 值 | 说明 |
|--------|-----|------|
| 高 | 1 | 紧急告警 |
| 中 | 2 | 普通告警 |
| 低 | 3 | 信息告警 |

**主要接口**:
| 方法 | 说明 |
|------|------|
| `CreateRule(userID, req)` | 创建告警规则 |
| `GetRule(ruleID, userID)` | 获取规则详情 |
| `ListRules(userID)` | 列出用户规则 |
| `UpdateRule(ruleID, userID, req)` | 更新规则 |
| `DeleteRule(ruleID, userID)` | 删除规则 |
| `EnableRule(ruleID, userID)` | 启用规则 |
| `DisableRule(ruleID, userID)` | 禁用规则 |
| `ProcessTelemetry(deviceID, data)` | 处理设备遥测数据，触发评估 |
| `GetAlerts(userID, status, limit, offset)` | 获取告警列表 |
| `AcknowledgeAlert(alertID, userID)` | 确认告警 |
| `ResolveAlert(alertID, userID)` | 解决告警 |

**数据表**:
| 表名 | 说明 |
|------|------|
| `alert_rules` | 告警规则表 |
| `alerts` | 告警记录表 |

**表达式语法**:
使用 [expr-lang/expr](https://expr-lang.org/) 语法。

可用变量:
- `temperature` - 温度
- `humidity` - 湿度
- `pressure` - 压力
- `battery` - 电量
- `status` - 状态 ("online"/"offline")

### 3.6 MQTT 消息服务 (MQTT Broker)

**模块**: `internal/mqtt/server.go`

**功能**:
- 内置 MQTT Broker (支持 MQTT 3.1.1/4.0 协议)
- 设备认证授权 (用户名=设备ID, 密码=设备密钥)
- 主题订阅/发布
- 遥测数据接收
- 心跳检测
- 命令下发
- **遥测数据回调 (触发告警评估)**

**MQTT 主题列表**:
| 主题 | 方向 | 描述 |
|------|------|------|
| `devices/{id}/telemetry` | 设备→平台 | 遥测数据上报 |
| `devices/{id}/heartbeat` | 设备→平台 | 设备心跳（可包含属性数据） |
| `devices/{id}/status` | 设备→平台 | 状态更新 |
| `devices/{id}/command` | 平台→设备 | 下发命令 |
| `devices/{id}/command/resp` | 设备→平台 | 命令响应 |

**认证流程**:
```
设备连接 → CONNECT报文(用户名+密码) → 验证设备ID和密钥 → 检查是否被禁用 → 认证成功/失败
```

**连接管理**:
- 已连接设备映射表 (`deviceConns`) 用于追踪设备连接
- 禁用设备时，平台发送 MQTT DISCONNECT 报文并关闭连接
- 被禁用的设备无法重新连接，直到被启用

- **用户删除保护** (有设备的用户显示删除按钮为禁用状态)

**设备类型选择交互**:
- 默认显示下拉框选择已有类型
- 点击"或输入新类型"切换为输入框
- 选择下拉框则隐藏输入框
- 输入新类型并保存后，自动创建设备类型

**设备类型删除保护**:
- 有设备的类型，Delete 按钮显示为禁用状态（半透明）
- API 也会返回错误 "该类型下有 X 个设备，无法删除"

**设备删除规则**:
- 只允许删除离线设备
- 在线设备必须先禁用再删除
- 禁用设备会立即断开其 MQTT 连接

**设备密钥获取流程**:
```
注册设备 → 自动弹出密钥窗口 → 复制 Device ID + Secret → 用于 MQTT 连接
```

```json

  "id": "设备唯一ID (UUID)",
```
