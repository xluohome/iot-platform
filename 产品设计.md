# IoT Platform 产品功能介绍

> ⚠️ 这是一个实验性项目，采用 vibe coding 方式开发。

## 1. 产品概述

IoT Platform 是一个轻量级的物联网平台，使用 Go 语言开发，集成了设备管理、MQTT 消息服务、实时 WebSocket 通信、数据存储和 Web 可视化仪表盘。

## 3. 核心功能模块

### 3.1 设备管理 (Device Manager)

**模块**: `internal/device/manager.go`

**功能**:
- 设备注册/注销
- 设备状态管理 (在线/离线)
- 设备属性更新
- 心跳处理
- 从数据库加载设备列表
- **自动创建设备类型（注册和更新时）**
- **设备禁用/启用（禁用后拒绝连接）**

**主要接口**:
| 方法 | 说明 |
|------|------|
| `Register(name, type, props)` | 注册新设备 |
| `Unregister(deviceID)` | 删除设备 |
| `GetDevice(deviceID)` | 获取设备信息 |
| `ListDevices()` | 列出所有设备 |
| `UpdateStatus(deviceID, status)` | 更新设备状态 |
| `UpdateDeviceInfo(deviceID, name, type)` | 更新设备信息 |
| `UpdateProperties(deviceID, props)` | 更新设备属性 |
| `HandleHeartbeat(deviceID)` | 处理设备心跳 |
| `DisableDevice(deviceID)` | 禁用设备（断开连接） |
| `EnableDevice(deviceID)` | 启用设备 |
| `GetStats()` | 获取设备统计 |

### 3.2 设备类型管理 (Device Types)

**模块**: `internal/storage/store.go`, `internal/api/server.go`

**功能**:
- 预设常用设备类型
- 设备类型 CRUD 操作
- 设备数量统计
- 类型使用校验（禁止删除有设备的类型）
- 注册设备时自动创建新类型

**预设设备类型**:
| 类型 | 说明 |
|------|------|
| sensor | 传感器 |
| actuator | 执行器 |
| gateway | 网关 |
| camera | 摄像头 |
| thermostat | 温控器 |
| light | 灯光 |
| switch | 开关 |
| meter | 计量器 |

### 3.3 MQTT 消息服务 (MQTT Broker)

**模块**: `internal/mqtt/server.go`

**功能**:
- 内置 MQTT Broker (支持 MQTT 3.1.1/4.0 协议)
- 设备认证授权 (用户名=设备ID, 密码=设备密钥)
- 主题订阅/发布
- 遥测数据接收
- 心跳检测
- 命令下发

**MQTT 主题列表**:
| 主题 | 方向 | 描述 |
|------|------|------|
| `devices/{id}/telemetry` | 设备→平台 | 遥测数据上报 |
| `devices/{id}/heartbeat` | 设备→平台 | 设备心跳（可包含属性数据） |
| `devices/{id}/status` | 设备→平台 | 状态更新 |
| `devices/{id}/command` | 平台→设备 | 下发命令 |
| `devices/{id}/command/resp` | 设备→平台 | 命令响应 |

**认证流程**:
```
设备连接 → CONNECT报文(用户名+密码) → 验证设备ID和密钥 → 检查是否被禁用 → 认证成功/失败
```

**连接管理**:
- 已连接设备映射表 (`deviceConns`) 用于追踪设备连接
- 禁用设备时，平台发送 MQTT DISCONNECT 报文并关闭连接
- 被禁用的设备无法重新连接，直到被启用

### 3.4 数据存储 (Storage)

**模块**: `internal/storage/store.go`

**功能**:
- SQLite 数据库存储
- 设备信息持久化
- 遥测数据存储
- 命令记录存储
- 设备类型存储
- 自动建表

**数据表**:
| 表名 | 说明 |
|------|------|
| `users` | 用户表 |
| `refresh_tokens` | Refresh Token 表 |
| `devices` | 设备表 |
| `telemetry_data` | 遥测数据表 |
| `device_commands` | 命令记录表 |
| `device_types` | 设备类型表 |

### 3.5 WebSocket 实时通信

**模块**: `internal/websocket/hub.go`

**功能**:
- 支持 WebSocket 长连接
- 广播消息到所有客户端
- 客户端连接/断开管理
- 实时推送设备状态更新

**事件类型**:
| 事件 | 说明 |
|------|------|
| `device_registered` | 新设备注册 |
| `device_unregistered` | 设备注销 |
| `device_updated` | 设备更新 |
| `mqtt_message` | MQTT消息 |

### 3.6 RESTful API

**模块**: `internal/api/server.go`

**功能**:
- 设备 CRUD 操作
- 设备类型 CRUD 操作
- 发送命令到设备
- 查询遥测数据
- 获取平台统计
- CORS 跨域支持

**设备 API 端点**:
| 方法 | 端点 | 描述 |
|------|------|------|
| GET | `/api/v1/devices` | 列出所有设备 |
| POST | `/api/v1/devices` | 注册新设备 |
| GET | `/api/v1/devices/:id` | 获取设备详情 |
| DELETE | `/api/v1/devices/:id` | 删除设备（仅离线设备） |
| PUT | `/api/v1/devices/:id` | 更新设备信息 |
| PUT | `/api/v1/devices/:id/properties` | 更新设备属性 |
| PUT | `/api/v1/devices/:id/disable` | 禁用设备 |
| PUT | `/api/v1/devices/:id/enable` | 启用设备 |
| POST | `/api/v1/devices/:id/command` | 发送命令 |
| GET | `/api/v1/devices/:id/telemetry` | 获取遥测数据 |
| GET | `/api/v1/devices/:id/commands` | 获取命令历史 |

**设备类型 API 端点**:
| 方法 | 端点 | 描述 |
|------|------|------|
| GET | `/api/v1/device-types` | 获取所有类型（含设备数量） |
| POST | `/api/v1/device-types` | 创建新类型 |
| PUT | `/api/v1/device-types/:id` | 更新类型 |
| DELETE | `/api/v1/device-types/:id` | 删除类型（有设备时禁止） |

**统计 API 端点**:
| 方法 | 端点 | 描述 |
|------|------|------|
| GET | `/api/v1/stats` | 获取统计信息 |
| GET | `/health` | 健康检查 |

### 3.7 用户认证 (Authentication)

**模块**: `internal/auth/`

**功能**:
- JWT Token 认证 (Access Token + Refresh Token)
- 用户注册和登录
- Token 自动刷新
- 用户登出
- 用户管理 (Admin)
- 设备所有权隔离

**认证端点**:
| 方法 | 端点 | 描述 |
|------|------|------|
| POST | `/api/v1/auth/register` | 用户注册 |
| POST | `/api/v1/auth/login` | 用户登录 |
| POST | `/api/v1/auth/refresh` | 刷新 Token |
| POST | `/api/v1/auth/logout` | 用户登出 |
| GET | `/api/v1/auth/me` | 获取当前用户 |

**用户管理端点** (Admin):
| 方法 | 端点 | 描述 |
|------|------|------|
| GET | `/api/v1/users` | 列出所有用户 |
| POST | `/api/v1/users` | 创建用户 |
| GET | `/api/v1/users/:id` | 获取用户详情 |
| PUT | `/api/v1/users/:id` | 更新用户角色 |
| PUT | `/api/v1/users/:id/disable` | 禁用用户 |
| PUT | `/api/v1/users/:id/enable` | 启用用户 |
| DELETE | `/api/v1/users/:id` | 删除用户 (有设备不能删除) |

**Token 配置**:
| 类型 | 有效期 |
|------|---------|
| Access Token | 30 分钟 |
| Refresh Token | 7 天 |

**用户角色**:
| 角色 | 权限 |
|------|------|
| admin | 管理所有用户和设备 |
| user | 仅管理自己的设备 |

**数据表**:
| 表名 | 说明 |
|------|------|
| `users` | 用户表 |
| `refresh_tokens` | Refresh Token 表 |

## 4. Web 界面

### 4.1 登录页面 (Login)

**文件**: `web/login.html`

**功能**:
- 用户登录
- 用户注册
- 登录后跳转到 Dashboard
- Token 存储和管理
- 显示当前用户信息

### 4.2 仪表盘 (Dashboard)

**文件**: `web/dashboard.html`

**访问要求**: 需要先登录

**功能**:
- 设备列表展示
- 设备状态统计
- **设备类型统计**
- **设备类型管理**
- 新增设备 (Admin 可分配设备所有者)
- 删除设备（仅离线设备）
- 查看设备密钥 (MQTT 认证凭证)
- **设备详情查看和编辑** (Admin 可修改所有者)
- **设备禁用/启用**
- 实时状态更新 (通过 WebSocket)
- 注册后自动弹出密钥提示
- **按设备类型筛选**
- **用户管理** (Admin 专属功能)
  - 用户列表
  - 创建/编辑/禁用/启用/删除用户
  - **用户删除保护** (有设备的用户不能删除)

**界面元素**:
- 统计卡片 (总设备数/在线/离线/类型数)
- 设备表格 (ID/名称/类型/**Owner**/状态/最后在线/操作)
- **用户管理 Tab** (Admin 专属)
- **设备类型管理弹窗** (类型列表/添加/编辑/删除)
- **设备卡片头部** (设备类型管理 + 新增设备按钮)
- 新增设备弹窗 (类型下拉选择/可输入新类型/**所有者选择**)
- 设备详情弹窗 (完整信息/属性编辑/类型选择交互/**所有者选择**/禁用启用按钮)
- 设备密钥查看弹窗 (显示 Device ID 和 Secret)
- 禁用状态显示 (设备列表和详情页)
- 用户管理弹窗 (创建/编辑/禁用/启用/删除)
- **用户删除保护** (有设备的用户显示删除按钮为禁用状态)

**设备类型选择交互**:
- 默认显示下拉框选择已有类型
- 点击"或输入新类型"切换为输入框
- 选择下拉框则隐藏输入框
- 输入新类型并保存后，自动创建设备类型

**设备类型删除保护**:
- 有设备的类型，Delete 按钮显示为禁用状态（半透明）
- API 也会返回错误 "该类型下有 X 个设备，无法删除"

**设备删除规则**:
- 只允许删除离线设备
- 在线设备必须先禁用再删除
- 禁用设备会立即断开其 MQTT 连接

**设备密钥获取流程**:
```
注册设备 → 自动弹出密钥窗口 → 复制 Device ID + Secret → 用于 MQTT 连接
```

## 5. 配置文件

**文件**: `config.yaml`

```yaml
server:
  http_port: 8080    # HTTP/WebSocket 端口
  ws_port: 8081      # (保留,实际共用8080)

mqtt:
  port: 1883         # MQTT Broker 端口
  host: "0.0.0.0"   # 监听地址

database:
  type: "sqlite"    # 数据库类型
  path: "iot.db"    # 数据库文件

device:
  heartbeat_interval: 30    # 心跳间隔(秒)
  offline_threshold: 90     # 离线阈值(秒)
```

## 6. 数据模型

### 6.1 设备 (Device)

**数据库存储模型** (Device):
```json
{
  "id": "设备唯一ID (UUID)",
  "name": "设备名称",
  "type_id": "设备类型ID (外键)",
  "status": "online/offline",
  "secret": "设备密钥 (MQTT认证用)",
  "disabled": "是否被禁用 (false=正常, true=禁用)",
  "properties": "设备属性 (JSON)",
  "last_seen": "最后在线时间",
  "created_at": "创建时间",
  "updated_at": "更新时间"
}
```

**API响应模型** (DeviceResponse):
```json
{
  "id": "设备唯一ID (UUID)",
  "name": "设备名称",
  "type_id": "设备类型ID",
  "type_name": "设备类型名称 (通过查询获得)",
  "user_id": "设备所有者ID",
  "owner_name": "设备所有者名称",
  "status": "online/offline",
  "secret": "设备密钥 (MQTT认证用)",
  "disabled": "是否被禁用",
  "properties": "设备属性 (JSON)",
  "last_seen": "最后在线时间",
  "created_at": "创建时间"
}
```

> **设计说明**: 设备表只存储 `type_id`，避免类型名称变更时的数据同步问题。API 响应通过 `ListDevicesWithTypes()` 和 `GetDeviceWithType()` 方法关联查询类型名称。`disabled` 字段用于控制设备是否允许连接。
