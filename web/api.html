Authentication (认证)

注意: 大部分 API 需要认证。请求时需要在 Header 中添加: Authorization: Bearer <access_token>

POST/api/v1/auth/register

用户注册

usernamestringrequired
passwordstringrequired
{ "username": "testuser", "password": "password123" }
POST/api/v1/auth/login

用户登录

usernamestringrequired
passwordstringrequired
{ "username": "admin", "password": "admin123" }

返回:

{ "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...", "refresh_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...", "expires_in": 1800, "token_type": "Bearer", "user": { "id": 1, "username": "admin", "role": "admin" } }
POST/api/v1/auth/refresh

刷新 Access Token

{ "refresh_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..." }
POST/api/v1/auth/logout

用户登出 (需要认证)

Header: X-Refresh-Token: <refresh_token>

GET/api/v1/auth/me

获取当前用户信息 (需要认证)

User Management (用户管理)

Admin only: 只有管理员可以访问这些接口

GET/api/v1/users

列出所有用户 (需要认证)

{ "users": [ { "id": 1, "username": "admin", "role": "admin", "disabled": false, "created_at": "2024-01-01T00:00:00Z" } ] }
POST/api/v1/users

创建用户 (需要认证)

usernamestringrequired
passwordstringrequired
rolestring
{ "username": "newuser", "password": "password123", "role": "user" }
GET/api/v1/users/:id

获取用户详情 (需要认证)

PUT/api/v1/users/:id

更新用户角色 (需要认证)

rolestring
{ "role": "admin" }
PUT/api/v1/users/:id/disable

禁用用户 (需要认证)

禁用后该用户无法登录

PUT/api/v1/users/:id/enable

启用用户 (需要认证)

DELETE/api/v1/users/:id

删除用户 (需要认证)

有设备的用户无法删除，会返回错误: "该用户拥有 X 个设备，无法删除"

Device Management

GET/api/v1/devices

List all registered devices (Admin sees all, users see only their own)

{ "devices": [ { "id": "uuid-string", "name": "Device Name", "type_id": 1, "type_name": "sensor", "user_id": 1, "owner_name": "admin", "status": "online/offline", "disabled": false, "secret": "device-secret-key", "properties": "{}", "last_seen": "2024-01-01T00:00:00Z", "created_at": "2024-01-01T00:00:00Z" } ], "total": 1 }
POST/api/v1/devices

Register a new device (Admin can specify user_id to assign to a user)

namestringrequired
typestringrequired
user_iduintAdmin only - assign device to user
propertiesobject
{ "name": "Temperature Sensor", "type": "sensor", "user_id": 2, "properties": {"location": "living room"} }

返回包含 secret，用于 MQTT 认证

GET/api/v1/devices/:id

Get device details by ID (includes secret)

PUT/api/v1/devices/:id

Update device information (name, type, user_id) - Admin can change owner

namestring
typestring
user_iduintAdmin only - reassign device owner
{ "name": "New Device Name", "type": "actuator", "user_id": 3 }

如果指定的类型不存在，会自动创建

PUT/api/v1/devices/:id/disable

Disable device (disconnect and prevent reconnection)

PUT/api/v1/devices/:id/enable

Enable device (allow reconnection)

PUT/api/v1/devices/:id/properties

Update device properties (Admin can also update owner via user_id)

{ "name": "New Name", "type": "sensor", "user_id": 2, "temperature": 25.5, "humidity": 60 }
DELETE/api/v1/devices/:id

Delete a device (only offline devices can be deleted)

仅允许删除离线设备，在线设备需先禁用

WebSocket Events

Connect: ws://localhost:8080/ws

The server broadcasts events to all connected clients. Event types:

// device_registered - New device was registered {"type": "device_registered", "payload": {...}} // device_unregistered - Device was deleted {"type": "device_unregistered", "payload": {"id": "uuid"}} // device_updated - Device status or properties changed {"type": "device_updated", "payload": {...}} // mqtt_message - Raw MQTT message received {"type": "mqtt_message", "payload": { "device_id": "uuid", "topic": "devices/xxx/telemetry", "payload": [...], "timestamp": "..." }}

MQTT Topics

Broker: tcp://localhost:1883

Authentication Required

MQTT connections require authentication using device credentials:

{ "username": "device_id (注册后的设备ID)", "password": "device_secret (设备密钥)" }

设备ID和密钥在注册设备时生成，可通过API获取。

Topic Direction Description
devices/{id}/telemetry Device → Server Sensor data (遥测数据)
devices/{id}/heartbeat Device → Server Keep-alive ping (心跳，包含属性数据)
devices/{id}/status Device → Server Status update (状态更新)
devices/{id}/command Server → Device Command to device (下发命令)
devices/{id}/command/resp Device → Server Command response (命令响应)
